Metrics is an *infographics generator* that provides a wide range of options to display statistics about **GitHub** accounts. With over 30 plugins and 300 options, it allows users to customize their metrics and render them as **SVG**, **Markdown**, **PDF**, or **JSON**. This tool can be used to generate metrics for **users**, **organizations**, and even **repositories**, making it a versatile solution for various use cases.

## Overview
Metrics is designed to be embedded everywhere, including **GitHub profile readmes**. It supports various types of accounts, providing a comprehensive overview of **GitHub** activity. The tool is highly customizable, with a range of plugins and options available to suit different needs.

## Features
Metrics offers a wide range of features, including:
* **Customizable**: With 47 plugins and 335 options, users can tailor their metrics to suit their needs.
* **Support for multiple account types**: Metrics can be used for **users**, **organizations**, and **repositories**.
* **Various output formats**: Metrics can be rendered as **SVG**, **Markdown**, **PDF**, or **JSON**.
* **Embeddable**: Metrics can be embedded everywhere, including **GitHub profile readmes**.
* **Plugins**: Metrics includes a range of plugins, such as the **isometric commit calendar** and **languages activity**.

### Plugins
Some of the notable plugins available for Metrics include:
* **Isometric commit calendar**: Provides a visual representation of commit activity.
* **Languages activity**: Displays information about programming languages used in **GitHub** repositories.
* **Full year calendar**: Shows a calendar view of commit activity for a full year.
* **Half year calendar**: Shows a calendar view of commit activity for a half year.

## Installation
To use Metrics, users can follow these steps:
1. Visit the [Metrics GitHub repository](https://github.com/lowlighter/metrics) to learn more about the tool and its usage.
2. Explore the available plugins and options to customize the metrics.
3. Use the Metrics tool to generate metrics for **users**, **organizations**, or **repositories**.
4. Embed the metrics in **GitHub profile readmes** or other locations as needed.

## Usage
Metrics can be used in a variety of ways, including:
* **GitHub profile readmes**: Embed metrics in **GitHub profile readmes** to showcase **GitHub** activity.
* **Repositories**: Use Metrics to generate metrics for **repositories** and track their activity.
* **Organizations**: Generate metrics for **organizations** to gain insights into their **GitHub** activity.
* **Users**: Use Metrics to generate metrics for **users** and track their **GitHub** activity.
